Intelligent Sensor Technology

Our sensors are designed to be smart, secure, and always ready, ensuring reliable performance even when the car is off.

üîã Low-Power Standby Mode

  • Consume minimal power (microamps) when idle.
  • Remain in a sleep state, monitoring for touch/proximity signals.
  • Powered by car's always-on $12V$ or $5V$ rail.

üß† Secure Wake-Up Logic

  • Sensor triggers a secure Microcontroller (MCU) upon detection.
  • MCU powers up sensor, captures/processes data, matches templates.
  • Sends secure signal to Body Control Module (BCM) or Engine Control Unit (ECU).
  • Process is fast (under 1 second), secure (encrypted), efficient.

üîê Key Security Features

  • Tamper Detection: Triggers alerts if bypass or damage attempts occur.
  • Encrypted Storage: Biometric templates stored securely in MCU memory, not the sensor itself.
  • Fail-Safe: System can fall back to key fob or PIN if sensor fails or is tampered with.

Multi-Biometric System Logic

Understanding how a multi-biometric system behaves, especially when some sensors are inside the car and initial access (door unlock) is required to reach them.

üß† Unlock the Door even After Fingerprint Fails?

Because:

  • Face recognition, voice recognition, and weight sensors are all inside the cabin.
  • If the door stays locked, the system cannot collect further biometric data from these in-cabin sensors.

Therefore, the system must:

  • Log the failed fingerprint attempt.
  • Allow conditional access (e.g., open door but restrict ignition).
  • Trigger in-cabin biometric checks immediately upon door opening.

Multi-Stage Authentication Flow

A clear, prioritized logic ensures secure access, adapting even to initial failures for optimal user experience.

‚úÖ Step-by-Step Access Flow

  1. Fingerprint at Door Handle (First Line of Access)
    • Match: Door unlocks, proceed to ignition authentication.
    • Fail: Door *temporarily* unlocks, partial cabin wake-up for in-cabin sensors.
  2. Door Unlocks Temporarily & In-Cabin Activation
    • Cabin sensors (Face, Voice, Pressure Mat) activate. User enters car.
  3. Final Authorization & Arbitration
    • Rule 1: Fingerprint Priority: If Fingerprint Match $\rightarrow$ ACCESS GRANTED.
    • Rule 2: In-Cabin Arbitration (if Fingerprint Fails):
      • Count Matches from Voice, Face, Pressure Mat.
      • $\ge 2$ Matches $\rightarrow$ ACCESS GRANTED.
      • $< 2$ Matches $\rightarrow$ ACCESS DENIED.

Seamless & Secure In-Driving Monitoring

Beyond initial access, the system continuously verifies the driver for enhanced safety and security during operation.

üöó Continuous Verification Protocol

  • Purpose: Continuous driver verification (e.g., via Steering Wheel Fingerprint, Driving Behavior patterns).
  • Failure Protocol During Driving:
    • If continuous verification fails at a particular point:
    • System prompts driver to re-authenticate within a certain time period (e.g., 5 minutes).
    • If re-authentication fails:
      • Inform that it's collecting data from sensors (e.g., recording in-cabin activities).
      • Go to fallback actions (giving alerts to user/tracking live location).

Robust Security & Contingency Plans

Layered safeguards prevent unauthorized use, and reliable fallback methods ensure owner access even in challenging scenarios.

üõ°Ô∏è Security Safeguards

  • Time-Limited Cabin Access: Defined window (e.g., 1 minute) to complete in-cabin authentication if initial fingerprint fails.
  • Ignition Lockout: Ignition is disabled until at least one in-cabin biometric successfully passes (contingent on arbitration).

üîë Fallback Mechanisms

  • Key Fob Proximity Unlock: Standard RFID/NFC technology for passive entry.
  • Mobile App Unlock: Secure authentication via a paired smartphone application.
  • Manual Key Override: A hidden physical key slot in the door handle as a last resort.

Immediate Actions on Access Denial

If authentication fails, the system triggers comprehensive security measures to protect the vehicle and alert the owner.

üö® Primary Actions

  • Ignition Lockout: The car‚Äôs ECU or BCM prevents the ignition system from activating, even if an unauthorized user is inside the car.
  • Admin Theft Alert: A real-time alert is sent to the registered admin via mobile app notification, SMS, email, or in-car telematics.
    • Alert includes: Time, location of failed attempt, which biometric(s) failed, and a snapshot from the in-cabin camera (if enabled and privacy regulations permit).
